# AgentOps Cloud

**The Enterprise Control Plane for AI Agents.**

Think Datadog + Okta + Cloudflare, for the hundreds of AI agents your company already has running and doesn't have a single place to see.

---

## The problem

As companies adopt AI, they end up with dozens or hundreds of agents scattered across OpenAI Agents SDK, LangGraph, CrewAI, AutoGen, n8n, MCP servers, and custom code — built by different teams, for different purposes, with no shared system of record. Nobody can answer, with confidence:

- **What AI agents do we actually have?**
- **What are they doing, and what do they cost?**
- **Who owns each one, and what can it access?**
- **What's the highest-leverage thing to fix first?**

AgentOps Cloud is that system of record — an inventory, risk, and optimization layer that sits above every framework instead of replacing any of them.

## What it does today (Phase 1 + 2 + 2c + Phase 4 blockchain layer)

- **Authenticates with OKX Wallet** — no passwords, no third-party identity provider. A challenge-response signature (never the address alone) creates or logs into your workspace and issues a normal session — the wallet itself is never touched again until you sign out. Provider-agnostic underneath: Google/Microsoft/GitHub/Okta/SAML are reserved as future sibling providers.
- **Runs an Enterprise Health Scan** — the centerpiece flow: choose a data source (upload a JSON/YAML agent manifest, or connect a GitHub repo), watch it discover, analyze, and reason over your fleet in real time, and get back a nine-section AI-generated Executive Report (executive summary, org overview, cost analysis, security risks, operational risks, optimization opportunities, business impact, priority actions, overall health score) plus a dedicated Optimization Plan sequencing every finding into Immediate Wins / 30-Day / 90-Day / Long-Term Architecture — with a deterministic fallback so it works with zero external config.
- **Discovers and lists every agent** in a workspace — framework, owner, status, cost, health, risk — in one table.
- **Surfaces real, explainable recommendations**: merge likely-duplicate agents, cut spend on top cost drivers, retire agents with no recent activity, flag agents holding high-risk permissions or no assigned owner, and suggest cheaper models. Every recommendation — and every Optimization Plan rating (priority, effort, risk, confidence) — is generated by an explicit, inspectable rule, never a black box.
- **Gives you a searchable activity timeline** and an at-a-glance overview (agents found, monthly cost, open risks, optimization opportunities).
- **Cryptographically seals every Executive Report** — once a Health Scan completes, its report is hashed and anchored on Base Sepolia (`EnterpriseReportRegistry`), and the report view shows a "Report Integrity" card with the on-chain record. No report content, org data, or agent data ever leaves the database — only a hash and small metadata reach the chain. Never blocks the scan: a chain failure just omits the card. See [`docs/SmartContracts.md`](docs/SmartContracts.md) and [`docs/VerificationGuide.md`](docs/VerificationGuide.md).
- **Pricing is real, even though it's free** — `GET /pricing` reads live from an on-chain `ServicePricing` contract (falling back to a static FREE default if the chain is unreachable), so every service correctly shows "FREE — future enterprise plans may introduce paid services" from one real source, not a hardcoded label. The payment-provider architecture (`PaymentProvider` ABC, ready for a future x402 adapter) exists but nothing charges anyone — see [`docs/FutureMonetization.md`](docs/FutureMonetization.md).
- **Ships a real, modular foundation** — not a demo. GitHub is a real, registered connector; LangGraph, CrewAI, MCP, Kubernetes, and cloud provider connectors, and an SDK for automatic agent discovery are architected in and land in later phases — see [`docs/Roadmap.md`](docs/Roadmap.md).

## What it deliberately isn't

No marketplace, no chat interface, no billing, no notification center, no vanity dashboards. Six pages, one job: give an enterprise a clear picture of its AI agent fleet and a ranked list of what to do about it. See [`docs/FutureVision.md`](docs/FutureVision.md) for what's explicitly out of scope.

## Quickstart

```bash
./scripts/bootstrap.sh   # one-time: install deps, migrate, seed sample data
./scripts/dev.sh          # every time: runs API (port 8000) + web (port 3000)
```

- Web app: http://localhost:3000
- API docs: http://localhost:8000/docs

No Docker, no Postgres, no Redis required — the API runs against a local SQLite file and auth is skipped (`AUTH_DISABLED=true` by default) so the dashboard loads directly with no wallet needed. Set it to `false` in both `apps/api/.env` and `apps/web/.env` (matching `SESSION_SECRET_KEY`/`SESSION_JWT_SECRET`) to exercise real OKX Wallet login. See [`docs/TechnicalDecisions.md`](docs/TechnicalDecisions.md) for why, and [`docs/Contributing.md`](docs/Contributing.md) for manual setup steps.

### Testing the Health Scan's GitHub connector

`app/services/connectors/github_adapter.py` does a real, static scan of a repo's **root-level** `requirements.txt` / `pyproject.toml` / `package.json` for known agent-framework package markers — it's a one-time heuristic, not real runtime usage tracking, and every agent it produces is tagged `needs_review: true`. Don't expect it to reflect actual agent activity, and don't expect a hit on a monorepo whose manifest lives in a subdirectory (root-only, by design).

Health Scan → New Scan → GitHub → paste `owner/repo` or the full URL. No token needed for these (60 unauthenticated GitHub API calls/hour is plenty for a handful of runs). Verified live against the adapter's exact logic:

| Framework | Repo | Detected via |
|---|---|---|
| LangGraph | `langchain-ai/react-agent` | `pyproject.toml` contains `langgraph` |
| LangGraph | `langchain-ai/langgraph-swarm-py` | same |
| CrewAI | `crewAIInc/crewAI` | `pyproject.toml` contains `crewai` |
| OpenAI Agents SDK | `openai/openai-agents-python` | `pyproject.toml` contains `openai-agents` |
| AutoGen | `disler/multi-agent-postgres-data-analytics` | `pyproject.toml` contains `pyautogen` |

Each produces exactly one detected agent, named `"{repo} ({framework})"`, with no cost/model/owner data. For the honest zero-hit path, scan `vinta/awesome-python` — no matching manifest, so the scan completes with 0 agents detected rather than faking a result.

## How it's built

```
Browser (Next.js 15, Vercel)
        │  same-origin — /api/v1/* rewritten server-side (next.config.ts)
        ▼
FastAPI /api/v1 (Railway)  →  Postgres in prod / SQLite in local dev (org-scoped)
        │                  ↘  Rule-based recommendation engine
        │  hash only, after a report is generated — never blocks the scan
        ▼
Base Sepolia: EnterpriseReportRegistry · ServicePricing · AgentOpsRegistry
```

Org-scoped from day one, versioned API, a connector adapter interface ready for real integrations, a provider-agnostic LLM abstraction (OpenAI today, others are a subclass away), and a matching `ChainProvider` abstraction for the on-chain trust layer (`app/services/chain/`). Full detail in [`docs/Architecture.md`](docs/Architecture.md) and [`docs/ContractArchitecture.md`](docs/ContractArchitecture.md).
